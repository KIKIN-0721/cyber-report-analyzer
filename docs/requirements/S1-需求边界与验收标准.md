# S1需求边界与验收标准

## 1. 文档目的

定义第一阶段（S1）开发边界，避免需求扩散，保障 2026-04-20 的 M1 验收可落地。

## 2. S1范围内必须完成

- 完成项目协同基线：分支、Issue、PR、CI 可运行。
- 完成首版规则台账 V0.1（按 P0/P1 分类）。
- 完成字段映射与样本设计文档。
- 完成解析与OCR链路的最小实现框架。
- 完成模型归一化协议与后端接口草案。

## 3. S1范围外内容

- 不做全量规则自动判定。
- 不做复杂权限管理与多角色体系。
- 不做高并发性能优化与集群部署。
- 不做模型效果调优与训练。

## 4. 输入与输出定义

输入：
- 样本 PDF 报告（原生 PDF 优先）
- 截图样本与规则清单条目

输出：
- 协同文档（S1执行手册、Issue清单、验收清单）
- 规则台账 V0.1
- 模型协议与后端接口草案
- 可运行源码骨架与基础测试

## 5. S1验收标准（DoD）

- P0 任务全部完成且可验证。
- P1 任务完成率不低于 80%。
- `develop` 或当前集成分支可通过 CI。
- 文档、代码、测试三者版本一致。
- 里程碑标签 `v0.1-m1` 可按计划发布。

## 6. 风险与控制

- 风险：任务并行导致文档和实现不一致。
- 控制：每日更新 Issue 状态，周日统一收敛到集成分支。
