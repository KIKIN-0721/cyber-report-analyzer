# S1后端接口草案

## 1. 目标

定义第一阶段最小可运行后端接口，支持任务提交、任务查询、结果查询、待复核查询和结果回填。

## 2. 接口列表（函数级草案）

1. `health()`
- 说明：健康检查
- 返回：`{"status": "ok"}`

2. `submit_report(file_name: str)`
- 说明：提交报告任务
- 输入：文件名字符串
- 返回：任务对象

3. `get_task(task_id: str)`
- 说明：查询任务
- 输入：任务 ID
- 返回：任务对象（不存在抛出 `KeyError`）

4. `set_task_result(task_id: str, result: dict)`
- 说明：回填分析结果并更新任务状态
- 输入：任务 ID、结果对象
- 返回：更新后的任务对象

5. `get_task_result(task_id: str)`
- 说明：查询任务结果对象
- 输入：任务 ID
- 返回：结果对象或 `null`

6. `get_pending_reviews(task_id: str)`
- 说明：查询待复核条目列表
- 输入：任务 ID
- 返回：待复核列表

## 3. 任务对象结构

```json
{
  "task_id": "task-0001",
  "file_name": "sample-report.pdf",
  "status": "queued | completed",
  "result": null
}
```

## 4. S1约束

- 先采用内存存储，不引入数据库依赖。
- 不做鉴权和权限控制。
- 不做并发和超时控制。
- 待复核接口仅返回内存中的 `REVIEW` 项，不引入独立工作流。

## 5. S2演进方向

- 接入 SQLite 存储任务与结果。
- 增加任务状态机（queued/running/completed/failed）。
- 增加分页查询和筛选接口。
