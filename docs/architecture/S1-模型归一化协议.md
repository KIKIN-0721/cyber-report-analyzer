# S1模型归一化协议

## 1. 目标

定义模型辅助模块在 S1 阶段的输入输出，统一语义表达，减少后续规则引擎适配成本。

## 2. 输入协议

输入类型：JSON 对象（字典）

建议字段：
- `field`：可选，原始字段名
- `value`：优先使用，原始识别值
- `raw_text`：可选，OCR原文
- `text`：可选，备用文本字段

示例：
```json
{
  "field": "crypto",
  "value": "RSA2048"
}
```

## 3. 输出协议

输出类型：JSON 对象（字典）

固定字段：
- `normalized`：归一化结果
- `explanation`：解释信息
- `confidence`：置信度字符串（0-1）

可选扩展字段：
- `policy_hint`：便于后端分流的策略提示，例如 `s1_rsa_review`

示例：
```json
{
  "normalized": "RSA-2048",
  "explanation": "RSA key length meets minimum baseline but should be reviewed by policy.",
  "confidence": "0.90",
  "policy_hint": "s1_rsa_review"
}
```

## 4. S1归一化规则

- RSA：支持 `RSA2048`、`2048-bit RSA` 等表达。
- TLS：支持 `TLS1.2`、`TLS 1.3` 等表达。
- 弱算法：识别 `MD5`、`SHA-1`、`DES`、`3DES`、`RC4`、`ECB`。
- 未命中规则：返回原文本并标记需要人工复核。

## 5. 错误处理

- 输入非字典：抛出 `TypeError`
- 输入为空：`confidence` 降低并返回人工复核提示
