# S1规则台账 v0.1

## 1. 说明

本台账用于第一阶段规则梳理，目标是先确定高频可判定规则，建立后续自动化基础。

## 2. P0规则（S1必须落地）

1. RSA 密钥长度识别
- 识别目标：`RSA-2048`、`RSA-3072` 等
- 初步判定：
  - `>=3072` 倾向 PASS
  - `2048` 倾向 REVIEW
  - `<2048` 倾向 FAIL

2. TLS 协议版本识别
- 识别目标：`TLS1.0`、`TLS1.1`、`TLS1.2`、`TLS1.3`
- 初步判定：
  - `1.2/1.3` 倾向 PASS
  - `1.0/1.1` 倾向 REVIEW/FAIL

3. 弱算法关键词识别
- 识别目标：`MD5`、`SHA-1`、`DES`、`3DES`、`RC4`、`ECB`
- 初步判定：命中即进入 REVIEW，优先人工复核

## 3. P1规则（S1文档化，S2/S3实现）

1. 端口服务与不安全配置关键词
2. 更新机制与签名机制检查项
3. 认证方式与默认口令相关条目

## 4. 字段标准化建议

- `field`: 规则字段名，如 `crypto.rsa.key_length`
- `value`: 提取值，如 `2048`
- `source_type`: `text` / `image_ocr`
- `page`: 来源页码
- `snippet`: 证据片段

## 5. S1输出要求

- 规则台账必须可追踪到来源文档。
- 每条规则必须定义：识别模式、判定倾向、复核策略。
- 每条规则必须标注优先级（P0/P1）。
