# S1 本地环境脚本说明

负责人：叶泽东  
阶段：S1  
目标：让新成员可以在本地快速完成最小开发环境初始化，并验证 parser MVP 能正常开发与测试。

## 1. 交付文件

- 脚本：`scripts/setup_local_env.ps1`
- 本说明：`deploy/S1-本地环境脚本.md`

## 2. 脚本能力

脚本默认完成以下动作：

1. 检查本机 Python
2. 创建或复用 `.venv`
3. 升级 `pip`
4. 安装最小测试依赖 `pytest`
5. 输出虚拟环境激活方式

可选动作：

- 执行 `python -m compileall src tests`
- 执行 `python -m pytest -q`

## 3. 使用方式

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup_local_env.ps1
```

如果要在初始化后立即做一次检查：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup_local_env.ps1 -RunChecks
```

如果想自定义虚拟环境目录：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup_local_env.ps1 -VenvPath .venv-s1
```

## 4. 初始化完成后的常用命令

激活虚拟环境：

```powershell
. .\.venv\Scripts\Activate.ps1
```

执行语法检查：

```powershell
python -m compileall src tests
```

执行测试：

```powershell
$env:PYTHONPATH='.'
python -m pytest -q
```

## 5. 适用范围

本脚本用于 S1 最小 MVP 开发环境，不处理以下内容：

- 生产部署
- Docker / 容器化
- 数据库初始化
- Web 服务上线

## 6. 验收标准

满足以下条件即可视为完成 S1 本地开发环境脚本初版：

- [ ] 新成员可一键创建虚拟环境
- [ ] `pytest` 可自动安装
- [ ] 可选执行基础检查
- [ ] 输出明确的下一步操作提示
